# 「Palette of Earth」Visual Direction

**마지막 업데이트:** 2026.05.29
**기반 문서:** `plan.md`, `ux.md`

---

## 1. 디자인 철학

> "자연은 이미 완성된 팔레트다. UI는 그것을 방해하지 않는다."

이 사이트의 주인공은 **자연이 만든 색**이다.
UI는 무대 조명처럼 색이 드러나는 순간을 연출하되, 스스로는 최대한 중립을 지킨다.
무겁지 않고, 자연스럽고, 탐험하는 즐거움이 있는 경험.

**키워드:** Earthy · Editorial · Cinematic · Restrained

---

## 2. 디자인 양식 (Design Language)

### 참조 레퍼런스 (직접 복사 아님, 분위기 참고)

| 레퍼런스 | 차용할 요소 |
|---------|-----------|
| Kinfolk Magazine | 여백, 사진 비중, 정적인 긴장감 |
| Are.na (디자인 리서치 도구) | 색이 오브젝트처럼 다루어지는 방식 |
| Letterboxd · A24 | 다크 배경 위 색이 빛나는 시각적 드라마 |
| 자연사박물관 디지털 아카이브 | 분류·발굴의 느낌, 라벨 텍스트 처리 |

### 전체 톤

- **다크 베이스**: 어두운 배경 위에서 자연색이 보석처럼 빛난다. 밝은 배경은 색을 희석시킨다.
- **고요한 긴장감**: 빠른 인터랙션 보다 느리고 의도적인 전환. 색이 "발굴"되는 느낌.
- **에디토리얼 밀도**: 글과 이미지가 동등한 무게를 가지는 레이아웃. 정보보다 인상 우선.

## 3. 컬러 시스템

### 베이스 팔레트 (UI 고정 색상)

| 역할 | 값 | 설명 |
|------|-----|------|
| Background | `#0D0C0B` | 숯보다 약간 따뜻한 검정. 순수 블랙보다 자연스럽다. |
| Surface | `#1A1917` | 카드·섹션 배경. 배경과 미묘하게 구분. |
| Border / Divider | `#2E2C28` | 선이 아닌 면 분리. 거의 보이지 않을 정도로. |
| Text Primary | `#F0EBE3` | 따뜻한 오프화이트. 순수 흰색은 지나치게 차갑다. |
| Text Secondary | `#8A8278` | 레이블, 캡션. 흙빛이 도는 중간 회색. |
| Text Tertiary | `#504D49` | 비활성 상태, 구분선 텍스트. |

### 지역 팔레트 (동적 색상 — 데이터에서 주입)

UI 자체의 색은 고정하지 않는다. 각 지역 섹션 진입 시 해당 팔레트의 색이 배경과 강조 요소에 스며든다.

**적용 규칙:**
- 배경 오버레이: 팔레트에서 가장 어두운 색 → `opacity: 0.08` (매우 서브틀하게)
- 강조 포인트 (active 도트, 하이라이트): 팔레트에서 가장 채도 높은 색
- 텍스트 색은 절대 바꾸지 않는다 (가독성 보장)

### 지역별 대표색 (색 칩 팔레트)

| 지역 | Tone 성격 | 팔레트 |
|------|----------|--------|
| 제주도 | 어둡고 강렬, 심해 | `#1A1A1A` `#1C3D5A` `#5C6B73` `#3D5C38` |
| 노르웨이 | 차갑고 깊은 청회 | `#1B2E3C` `#3B5E6E` `#6B8A7A` `#C8D4D0` |
| 모로코 | 따뜻한 흙, 황토 | `#8B4513` `#C17B3E` `#D4A96A` `#9E8B7A` |
| 뉴질랜드 | 청명하고 신선한 청록 | `#1A4A5C` `#2E7D6B` `#5AADA0` `#A8D5D0` |
| 아이슬란드 | 화산과 빙하, 극적 | `#1C1C1E` `#2E4A4E` `#6A8E8A` `#D8E8E4` |

> 실제 자연 사진에서 Canvas API로 추출한 값으로 교체 예정. 위 값은 개발 초기 플레이스홀더.

## 4. 타이포그래피

### 서체 선택

| 역할 | 서체 | 이유 |
|------|------|------|
| Heading (영문) | **Cormorant Garamond** (Italic 포함) | 자연사 아카이브의 에디토리얼 감성. 가느다란 세리프가 색과 대비된다. |
| Heading (한글) | **Noto Serif KR** | Cormorant와 같은 에디토리얼 계열의 한글 세리프. |
| Body / UI | **DM Sans** (영문) + **Pretendard** (한글) | 명료하고 현대적. 레이블, 설명 텍스트, UI 요소. |
| Label / Tag | **DM Mono** 또는 **JetBrains Mono** | HEX 값, RGB 수치, 지역 코드 등 데이터성 텍스트에 사용. |

### 타이포그래피 스케일

```
Display (Hero 타이틀)   : 72–96px / Cormorant Garamond Italic / line-height 1.0
H1 (지역명)             : 48–64px / Cormorant Garamond Regular / line-height 1.1
H2 (자연 요소명)        : 24–32px / DM Sans Medium / line-height 1.2
Body                   : 16–18px / DM Sans Regular / line-height 1.6
Caption / Label        : 12–13px / DM Sans Regular / letter-spacing 0.08em
Mono (HEX/RGB)         : 12–14px / DM Mono / letter-spacing 0.04em
```

### 텍스트 처리 원칙

- **영문 제목은 소문자 또는 Sentence case** → 대문자 올캡스는 사용하지 않는다. 너무 경직된 느낌.
- **지역명은 원어 병기** → "제주도 / Jeju" 형태로 한글+영문 Italic 조합
- **설명 문장은 1–2줄, 여백으로 숨 쉬게** → 긴 본문 블록 없음
- **HEX/RGB 값은 Mono 폰트로** → 데이터임을 시각적으로 명확히

## 5. 간격 & 레이아웃 시스템

### 기본 단위

- **8px 그리드** 기반
- 컴포넌트 내부 padding: 16px / 24px / 32px
- 섹션 간 여백: 최소 80px (모바일) / 120–160px (데스크톱)
- 컨텐츠 최대 너비: 1440px (마진 오토)

### 레이아웃 원칙

- **여백이 언어다.** 요소를 추가하기 전에 여백을 제거하는 것을 먼저 시도한다.
- **비대칭 그리드 활호.** 지역 섹션은 텍스트 40% / 이미지 60% 비대칭 분할. 정렬보다 긴장감.
- **세로 리듬.** 텍스트 블록과 색 칩 사이에 일관된 수직 간격 유지 (32px).

### 컬러 칩 레이아웃

```
데스크톱: 칩 4개 가로 배열, 간격 16px, 각 칩 52×52px
모바일:  칩 4개 가로 배열 유지 (크기 40×40px), 간격 12px
칩 아래: HEX 값 Mono 폰트, text-align center, 12px
```

## 6. 이미지 정책

### 선정 기준

- **비가공 자연 사진만** 사용. 스튜디오 촬영, 일러스트, AI 생성 이미지 사용 금지.
- **특정 자연 요소에 집중한 클로즈업** 우선. 풍경 전체보다 질감·형태가 중요.
  - 예: 현무암 표면, 피오르드 수면, 사막 모래 입자, 용암 지형
- **자연광 촬영만** 사용. 플래시, 인위적 조명 없음.
- **저작권 무료 소스:** Unsplash, Pexels (출처 표기 확인 필요)

### 이미지 처리 규칙

| 규칙 | 내용 |
|------|------|
| 리사이즈 | 레이어용 이미지: 최대 1200×900px (2레이어 × 5지역 = 10장) |
| 포맷 | WebP 우선, JPEG fallback |
| 비율 | 16:9 또는 4:3 고정 (레이어 정확한 정렬을 위해) |
| 컬러 프로파일 | sRGB (Canvas API 색 추출 일관성) |
| Blend 조합 | 레이어1(암석·토양) + 레이어2(물·식생) — 대비가 큰 두 자연 요소 쌍 |

### 이미지가 없을 때 (Fallback)

- 팔레트 4색으로 만든 수직 그라디언트 블록을 placeholder로 사용
- `aspect-ratio: 4/3` 유지

## 7. 모션 & 애니메이션 언어

### 원칙

- **느리고 의도적.** 기본 transition duration은 0.6–1.2s. 빠른 튀어나오기 없음.
- **이징은 자연 물리.** `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo 계열) 기본값.
- **페이드 우선.** 요소는 슬라이드보다 페이드로 등장한다. 슬라이드는 8–16px 이내 미세 이동에만 사용.
- **하나씩 드러내기.** 색 칩은 동시에 나타나지 않는다. 0.15–0.2s 간격으로 순차 등장.

### Easing 레퍼런스

```js
const easing = {
  enter: [0.16, 1, 0.3, 1],       // 자연스럽게 안착
  exit:  [0.7, 0, 0.84, 0],       // 빠르게 사라짐
  color: [0.4, 0, 0.2, 1],        // 배경색 전환 (MUI 기본 easing)
};
```

### `prefers-reduced-motion` 대응

모든 애니메이션에 적용. 최종 상태를 즉시 표시하고 전환 없음.

## 8. 섹션 구분 방식

### 원칙: 선으로 나누지 않는다

`<hr>`, border, divider 라인 사용 금지. 섹션은 다음 방법으로만 구분한다.

| 방법 | 사용 상황 |
|------|---------|
| **배경색 미세 전환** | 지역 섹션 간 전환 (0.8s ease) |
| **여백** | 같은 배경색 내 콘텐츠 블록 구분 |
| **이미지 블록** | 이미지 자체가 섹션 경계 역할 |
| **타이포 계층** | 지역명 대형 타이틀이 새 섹션 시작을 알림 |

### 섹션 진입 시각 처리

- 지역 섹션 진입: 지역명이 fade-up으로 먼저 등장 → 이미지 블렌드 → 팔레트 칩
- 배경은 섹션이 50% 진입했을 때 이전 지역에서 다음 지역 색으로 전환
- Hero → 첫 지역: 배경이 `#0D0C0B`에서 제주도 오버레이 색으로 서서히 스며듦

## 9. UI 컴포넌트 비주얼 가이드

### 컬러 칩

```
기본:  52×52px, border-radius 8px, box-shadow 0 2px 12px rgba(0,0,0,0.4)
       (다크 배경이므로 그림자가 더 강해야 칩이 떠있는 느낌)
Hover: scale 1.12, 위 8px 이동(translateY -8px), 툴팁 등장
Click: scale 0.96 → 1 (눌리는 피드백), 체크 아이콘 0.4s
```

### ProgressNav (우측 도트)

```
도트: 6px 원, 비활성 #504D49
활성: 20px 높이의 얇은 세로 바(캡슐형), 지역 대표색
전환: width+height 0.3s ease
```

### ColorToast

```
위치: 화면 하단 중앙, bottom 32px
형태: pill shape, background #1A1917, border 1px solid #2E2C28
텍스트: "복사됨" (DM Mono, 13px) + HEX값
등장: slide-up 0.3s + 2초 후 fade-out
```

### Globe (Hero)

```
배경: 완전 투명 Canvas, 페이지 배경색 그대로 보임
구체: 어두운 ocean 텍스처 또는 #1A1917 단색 + 와이어프레임 격자 (subtle)
지역 핀: 8px 원, 팔레트 1번 색
활성 핀: pulse 링 애니메이션 (scale 1→2, opacity 1→0, 1.5s loop)
```

## 10. 금지 사항 (What to Avoid)

| 금지 | 이유 |
|------|------|
| 밝은 배경(흰색/연회색) | 자연색의 드라마가 사라짐 |
| 과도한 그라디언트 | 자연 사진과 충돌 |
| 둥근 모서리 과용 (>12px) | 에디토리얼 긴장감 저해 |
| 색 칩에 텍스트 오버레이 | 칩은 순수한 색 블록이어야 함 |
| 아이콘 남발 | 텍스트와 색으로만 내비게이션 가능하게 |
| 빠른 바운스 애니메이션 | 브랜드 톤과 불일치 |
| 순수 블랙(`#000`) / 순수 화이트(`#FFF`) | 따뜻한 베이스와 충돌 |

## 11. 검수 기준

비주얼 디렉션에 맞는지 확인할 체크리스트:

- [ ] 자연 사진 위에서도 텍스트가 읽히는가 (대비비 4.5:1)
- [ ] 색 칩 4개가 배경 대비 충분히 선명하게 보이는가
- [ ] 섹션 전환 시 경계가 '선'으로 보이지 않는가
- [ ] 타이틀 세리프체와 UI 산세리프체가 자연스럽게 공존하는가
- [ ] 애니메이션이 없어도 (reduced-motion) 콘텐츠가 완전히 전달되는가
- [ ] HEX 값은 모노스페이스 폰트로 표시되는가
- [ ] 모바일에서 색 칩 40px이 탭하기 충분한가 (최소 터치 44px 권장)
