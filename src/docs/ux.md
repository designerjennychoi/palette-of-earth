## 1. 사용자 경험 목표

| 목표 | 측정 기준 |
|------|----------|
| 색 추출 과정을 직관적으로 이해 | 인터랙션 없이도 흐름이 읽히는가 |


## 4. 섹션별 UX 상세

### 4-1. Hero 섹션

**레이아웃**
- 전체 뷰포트(100vh) 차지
- 지구본 중앙 배치, 좌측 또는 하단에 CTA 문구

**인터랙션 상태**

| 상태 | 트리거 | 동작 |
|------|--------|------|
| Idle | 진입 직후 | 지구본 천천히 자동 회전 (y축, ~30초/바퀴) |
| Hover | 지역 핀/영역 위에 마우스 | 해당 지역 팔레트 1번 색으로 지구본 부분 하이라이트 + 지역명 툴팁 |
| Active | 지역 클릭 | 지구본 전체 팔레트 색으로 전환 (2–3초 fade) → 해당 섹션 스크롤 |

**CTA**
- 텍스트: "지역의 색을 만나다" (1차) / 스크롤 유도 아이콘 (chevron-down)
- 클릭 시 첫 번째 지역 섹션으로 스무스 스크롤

**진입 애니메이션 (초기 로드)**
1. 배경 fade-in (0 → 0.3s)
2. 지구본 scale-up + fade-in (0.3 → 1.0s)
3. CTA 텍스트 fade-up (0.8 → 1.3s)

---

### 4-2. 지역 섹션 (×5)

**레이아웃 (데스크톱)**
```
┌─────────────────────────────────────────┐
│  지역명         이미지 블렌드 영역        │
│  자연 요소      (60% 너비)               │
│                                          │
│  컬러 팔레트                             │
│  ○ ○ ○ ○                                │
│                                          │
│  설명 문장                               │
└─────────────────────────────────────────┘
```

**레이아웃 (모바일)**
- 이미지 블렌드 → 상단 전체 너비
- 텍스트 + 팔레트 → 하단 스택

**이미지 블렌드 애니메이션 시퀀스**

| 단계 | 타이밍 | 동작 |
|------|--------|------|
| 0 | 뷰포트 진입 전 | 레이어1 opacity 0 |
| 1 | 진입 후 0 → 0.6s | 레이어1(토양/자연물) fade-in (opacity 0→1) |
| 2 | 0.6 → 1.4s | 레이어2(물/식생) overlay, mix-blend-mode: multiply, opacity 0→0.7 |
| 3 | 1.4 → 2.0s | 두 레이어 안정화, 팔레트 영역으로 시선 유도 |
| 4 | 2.0 → 2.8s | 컬러 칩 4개 순차 fade-up (0.2s 간격) |

- Intersection Observer 기반으로 뷰포트 진입 시 1회 트리거
- 이미 재생된 섹션은 상태 유지 (다시 진입해도 재생 안 함)

**배경색 연동**
- 스크롤 위치가 해당 지역 섹션 50% 이상 진입 시 전환
- 배경색: 해당 팔레트의 가장 밝은 색, opacity 0.12 (서브틀하게)
- 전환 duration: 0.8s ease-in-out

---

### 4-3. 컬러 칩 인터랙션

**기본 상태**
- 원형 또는 정사각형(모서리 12px radius)
- 크기: 데스크톱 52×52px / 모바일 40×40px
- 그림자: `box-shadow: 0 2px 8px rgba(0,0,0,0.15)`

**Hover 상태**
- scale: 1 → 1.15 (0.2s ease)
- 칩 위에 툴팁 등장: HEX + RGB 값
- 툴팁 위치: 칩 상단 중앙

**Click/Tap 상태**
- 클립보드에 HEX값 복사
- 피드백: 칩 내부에 체크 아이콘 0.5s 표시 후 원복
- 하단 스낵바/토스트: "HEX 복사됨 #XXXXXX" (2초 후 자동 소멸)

---

### 4-4. 이미지 호버 인터랙션

**동작**
- 이미지 위 마우스 진입 → 해당 이미지에서 추출된 컬러 칩에 pulse 링 강조
- 이미지와 칩 사이 연결선 또는 하이라이트 배경으로 시각적 연결 표시
- 모바일: 롱프레스(500ms) 또는 탭으로 대체

---

### 4-5. 색 비교 섹션 (Optional)

**레이아웃**
```
┌─────────────────────────────────────────┐
│  "같은 색, 다른 땅"                       │
│                                          │
│  [모로코 사막]      [아이슬란드 화산재]    │
│  ████ ████          ████ ████            │
│                                          │
│  "왜 둘 다 따뜻한 갈색-회색 톤일까?"       │
└─────────────────────────────────────────┘
```

- 두 지역 팔레트를 나란히 배치, 같은 톤 칩에 선으로 연결
- 진입 시 연결선 draw 애니메이션 (SVG stroke-dashoffset)

## 5. 내비게이션 & 스크롤

### 진행 표시 (Progress Indicator)
- 우측 고정 세로 도트 내비게이션 (Hero + 5개 지역 + Optional 비교)
- 현재 섹션 도트: 활성색으로 강조
- 도트 호버: 지역명 툴팁
- 도트 클릭: 해당 섹션으로 스무스 스크롤

### 스크롤 스냅 (선택적)
- 각 지역 섹션에 `scroll-snap-align: start` 적용 검토
- 애니메이션 시퀀스 완료 전 스냅 이탈 방지 고려

## 6. 반응형 브레이크포인트

| 브레이크포인트 | 너비 | 주요 변경 |
|--------------|------|---------|
| Mobile | < 768px | 이미지 상단 배치, 팔레트 칩 하단 스택, 지구본 80vw |
| Tablet | 768–1199px | 2컬럼 레이아웃, 팔레트 인라인 |
| Desktop | ≥ 1200px | 기본 레이아웃 (좌텍스트 + 우이미지) |

---

## 7. 접근성 고려사항

- 컬러 칩에 `aria-label="색상 이름, HEX값, 클릭하면 복사"` 제공
- 지구본 인터랙션에 키보드 탭 내비게이션 지원 (지역 선택 가능)
- 이미지 블렌드 애니메이션: `prefers-reduced-motion` 미디어쿼리 대응 → 애니메이션 생략, 최종 상태 즉시 표시
- 배경색 변경 시 텍스트 대비비 4.5:1 이상 유지 확인

## 8. 빈 상태 & 엣지 케이스

| 상황 | 처리 |
|------|------|
| 이미지 로딩 실패 | 팔레트 색상으로 채운 placeholder 블록 표시 |
| 클립보드 접근 거부 | 토스트에 "브라우저에서 복사를 허용해주세요" 표시 |
| 터치 디바이스 | hover 인터랙션 → tap 전환, 롱프레스 대체 |
| 느린 네트워크 | 이미지 lazy-load + blur placeholder |
